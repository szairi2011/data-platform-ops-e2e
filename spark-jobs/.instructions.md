## Four Jobs

### 1. BronzeIngester
- **Input**: raw CSV/JSON from HDFS `hdfs:///data/raw/`
- **Output**: Parquet on HDFS `hdfs:///data/bronze/` partitioned by `ingest_date`
- **Key behaviours**:
  - Schema enforcement via `StructType` — malformed rows go to `hdfs:///data/dead-letter/`
  - Idempotent: `SaveMode.Overwrite` on partition (not full table)
  - Emits OpenLineage `START`/`COMPLETE` events

### 2. SilverTransformer
- **Input**: Bronze Parquet `hdfs:///data/bronze/`
- **Output**: Silver Parquet `hdfs:///data/silver/` partitioned by `event_date`
- **Key behaviours**:
  - Incremental: reads only new partitions not yet in Silver (watermark via Airflow Variable `SILVER_WATERMARK`)
  - Broadcast join for small dimension tables (< 10 MB)
  - Partition pruning: filter pushed down before joins
  - Emits OpenLineage events

### 3. GoldAggregator
- **Input**: Silver Parquet `hdfs:///data/silver/`
- **Output**: Hive external table `gold_db.daily_summary` on `hdfs:///data/gold/`
- **Key behaviours**:
  - Creates Hive database + table via SparkSQL DDL if not exists
  - Atomic write: write to temp dir, then `ALTER TABLE ... SET LOCATION`
  - Output partitioned by `event_date`
  - Emits OpenLineage events

### 4. CompactionJob
- **Input**: any Parquet layer (configurable via `app.compaction.layer`)
- **Output**: same path (in-place compaction)
- **Key behaviours**:
  - Reads partition, coalesces to target file count (`app.compaction.targetFiles`), overwrites
  - Uses `coalesce` when reducing partitions (no shuffle), `repartition` when increasing
  - Safe for maintenance windows only — not part of main pipeline DAG

## Cross-Cutting Requirements

- **Single SparkSession**: `SparkSessionFactory.getOrCreate(appName)` — never create inline
- **Typesafe Config**: all HDFS paths, Hive DB names, tuning params in `application.conf`
- **No cluster dependency in unit tests**: use `SparkSession.builder.master("local[2]")` in tests
- **Assembly**: shade all dependencies; mark `spark-core`, `spark-sql`, `hadoop-client` as `provided`
- **OpenLineage**: `SparkContext.setLocalProperty("openlineage.parentJobName", ...)` before each action

## CI/CD (GitHub Actions)

Two-job workflow in `.github/workflows/ci.yml`:

**Job 1 — `test-and-build`** (GitHub-hosted runner, `ubuntu-latest`):
1. `sbt test` — unit tests only, no cluster
2. `sbt assembly` — produces `target/scala-2.12/spark-jobs-assembly*.jar`
3. Upload JAR as Actions artifact (`spark-jobs-assembly`, 7-day retention)

**Job 2 — `deploy`** (self-hosted runner — same machine running Docker YARN cluster):
- Runs only on `push` to `main` (not PRs)
- Downloads JAR artifact from job 1
- `docker cp <jar> namenode:/tmp/` → `docker exec namenode hdfs dfs -put -f`
- Verifies JAR on HDFS: `hdfs dfs -ls hdfs:///jars/spark-jobs-assembly.jar`
- `PATCH /api/v2/variables/JAR_HDFS_PATH` (Airflow 3.x REST API)
- `PATCH /api/v2/variables/JAR_VERSION` = `${{ github.sha }}`

**Self-hosted runner setup (once)**:
- Register runner for `szairi2011/spark-jobs` with label `self-hosted`
- Runner process must be on the same machine as the Docker YARN cluster
- Set GitHub secret `AIRFLOW_PASSWORD` (admin user password)
- Optionally set `vars.AIRFLOW_BASE_URL` if not `http://localhost:8080`

**HDFS JAR path strategy**: fixed path `hdfs:///jars/spark-jobs-assembly.jar` (overwrite on each deploy).
`JAR_VERSION` variable = git SHA for audit trail. Airflow DAGs always read `JAR_HDFS_PATH`.

## SDD Rules

1. No Spark job without a corresponding ScalaTest spec (no-cluster, `local[2]`)
2. No hardcoded HDFS paths — always via `AppConfig`
3. All jobs must accept `--env dev|staging|prod` arg (switches `application.conf` profile)
4. OpenLineage emission is mandatory for Bronze, Silver, Gold (not Compaction)
