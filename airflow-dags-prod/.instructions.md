## Airflow 3.x Architecture Notes

- Airflow 3.2.1 ships a **DAG processor** as a separate process — include it in docker-compose
- The **Task SDK** is used for all new task definitions (not legacy operators where possible)
- `SparkSubmitOperator` remains available via `apache-airflow-providers-apache-spark`
- Webserver, scheduler, dag-processor, and workers are separate services
- Default executor: `CeleryExecutor` (Redis broker + Postgres result backend)

## DAGs

### medallion_pipeline (daily, 02:00 UTC)

```
bronze_ingest → silver_transform → gold_aggregate
                      ↓                  ↓
              ge_validate_silver   ge_validate_gold
```

- Each Spark task uses `SparkSubmitOperator` (deploy-mode: cluster, master: yarn)
- JAR path read from Airflow Variable `JAR_HDFS_PATH`
- `--conf spark.yarn.appMasterEnv.PIPELINE_ENV={{ var.value.PIPELINE_ENV }}`
- `silver_transform` writes `SILVER_WATERMARK` via XCom on success
- `ge_validate_silver` / `ge_validate_gold` → `BashOperator` calling Great Expectations CLI

### compaction (weekly, Sunday 03:00 UTC)

```
compaction_bronze → compaction_silver → compaction_gold
```

- Each task calls `SparkSubmitOperator` with `--conf app.compaction.layer=bronze|silver|gold`

---

## Docker Compose

- All Airflow containers join **`spark-net`** (external network — created by yarn-cluster stack)
- Hadoop client config (`core-site.xml`, `yarn-site.xml`) bind-mounted into worker at `/opt/hadoop/conf/`
- `HADOOP_CONF_DIR=/opt/hadoop/conf` set in environment
- `SPARK_HOME` set to the Spark installation inside the custom image

---

## SDD Rules

1. No DAG without a DAG import test in `tests/test_dag_integrity.py`
2. All `SparkSubmitOperator` tasks must set `deploy_mode="cluster"` and `master="yarn"`
3. JAR path always from Variable `JAR_HDFS_PATH` — never hardcoded
4. Airflow must NOT call GitHub Actions or any CI/CD endpoint
5. `medallion_pipeline` must be idempotent: re-running the same `logical_date` is safe
6. Use Airflow 3.x `logical_date` (not deprecated `execution_date`)
