# yarn-cluster — SDD Spec

## Purpose

New standalone repo (`szairi2011/yarn-cluster`) that provides the Hadoop YARN + Spark
compute layer for the platform. Built from scratch using the same image versions as
`learn-jupyterhub-with-livy` as the version reference — do NOT copy code from that repo.

Copilot must read this file before editing any file in `yarn-cluster/`.

---

## What This Submodule Provides

| Service | Host (container name) | Port(s) |
|---------|----------------------|---------|
| Hadoop NameNode | `namenode` | 9000 (HDFS), 9870 (UI) |
| Hadoop DataNode × 2 | `datanode1`, `datanode2` | 9864 (UI) |
| YARN ResourceManager | `resourcemanager` | 8032 (API), 8088 (UI) |
| YARN NodeManager × 2 | `nodemanager1`, `nodemanager2` | 8042 (UI) |
| Apache Livy | `livy` | 8998 |
| Hive Metastore | `hive-metastore` | 9083 (Thrift) |
| PostgreSQL (HMS) | `postgres` | 5432 |
| JupyterHub | `jupyterhub` | 8000 |

Network: `spark-net` (Docker bridge)

## How Other Stacks Reach the Cluster

- **Airflow → YARN**: `spark-submit --master yarn` — Airflow containers join `spark-net`
- **Airflow → HDFS**: `hdfs://namenode:9000` — namenode resolvable on `spark-net`
- **Spark → Hive**: `hive.metastore.uris=thrift://hive-metastore:9083`
- **OpenLineage → OpenMetadata**: Spark jobs emit to `http://openmetadata:4343`


`init-hdfs.sh` runs once inside `namenode` container via a Docker healthcheck dependency.

---

## Spark Event Log (required for job monitoring)

`config/livy/spark-defaults.conf` must include:
```
spark.eventLog.enabled    true
spark.eventLog.dir        hdfs://namenode:9000/user/spark/spark3ApplicationHistory
spark.history.fs.logDirectory  hdfs://namenode:9000/user/spark/spark3ApplicationHistory
```

## Key Design Decisions

- `spark-net` is **external** in this compose — other stacks join it by name
- NodeManager images include Python 3.10 + PySpark to support YARN cluster-mode Python drivers
- JupyterHub uses `DummyAuthenticator` (local dev only) + `DockerSpawner` (singleuser containers)
- Hive Metastore uses `apache/hive:4.0.0` directly with PostgreSQL JDBC — no custom Dockerfile needed
- Livy configured for both interactive sessions (SparkMagic) and batch submissions (SparkSubmitOperator)

---

## SDD Rules

1. All container names are **stable service-discovery hostnames** — do not rename them
2. `spark-net` name is fixed — airflow-dags-prod and data-governance depend on it
3. No business logic — this repo is pure infrastructure
4. Hadoop config files must be mounted (not baked into images) for easy tuning
5. `init-hdfs.sh` must be idempotent (`hdfs dfs -mkdir -p`, not raw `mkdir`)
